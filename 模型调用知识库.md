
## 轻代码构建RAG知识库（AI专业加分，贴合课程作业，少量Python代码）
### 核心工具
**LangChain（开源框架，负责数据处理/检索）+ Chroma（轻量级向量数据库，负责存储数据）**，均免费，纯本地运行，不用服务器。
### 前置准备
1. 电脑安装Python（3.8及以上，之前提过的，免费）；
2. 安装依赖包（打开CMD/终端，输入**一行命令**即可，复制粘贴运行）：
   ```
   pip install langchain chroma-langchain pypdf pandas openpyxl
   ```
   （该命令会自动安装构建知识库所需的所有工具，不用单独装）
### 具体步骤（贴合服装电商，代码极简，不用懂底层）
#### 步骤1：整理服装数据（和零代码一样，3类核心数据，整理成**纯文本TXT**或Excel，方便代码读取）
#### 步骤2：写**极简Python代码**（核心代码仅10几行，不用改逻辑，只需替换你的「数据文件路径」）
代码核心做3件事：
1. 读取你整理的服装数据文件（TXT/Excel）；
2. 把数据拆分成小片段，转换成AI能检索的「向量格式」；
3. 把向量数据存入Chroma本地数据库，生成RAG知识库。
**代码思路（不用写具体代码，照这个框架复制粘贴即可）**：
```python
# 1. 导入所需工具
from langchain.document_loaders import TextLoader/ExcelLoader  # 读取数据文件
from langchain.text_splitter import CharacterTextSplitter    # 拆分数据
from langchain.vectorstores import Chroma                     # 存储数据
from langchain.embeddings import DashScopeEmbeddings         # 生成向量（通义千问免费embedding）

# 2. 读取你的服装数据文件（替换成你的文件路径，比如C:/服装数据.txt）
loader = TextLoader("你的服装数据文件路径")
documents = loader.load()

# 3. 拆分数据（平台自动设置参数，不用改）
text_splitter = CharacterTextSplitter(chunk_size=100, chunk_overlap=20)
texts = text_splitter.split_documents(documents)

# 4. 生成向量并存入Chroma，创建RAG知识库（替换成你的通义千问API-Key，免费）
embeddings = DashScopeEmbeddings(api_key="你的通义千问API-Key")
db = Chroma.from_documents(texts, embeddings, persist_directory="./服装知识库")
db.persist()  # 保存知识库到本地，下次直接调用
```
#### 步骤3：对接你的智能体大模型（代码调用）
在你的智能体代码中，添加**「检索知识库」逻辑**（仅几行代码），实现：用户提问→代码从本地服装知识库中检索精准答案→把答案传给大模型→大模型结合答案生成贴合服装场景的回复。
**核心调用思路**：
```python
# 从本地服装知识库中检索相关内容
query = "160/90斤穿什么码？"  # 用户的问题

# 把检索到的答案传给大模型，生成回复
prompt = f"根据以下服装知识回答问题：{docs[0].page_content}，问题：{query}"
# 调用大模型API（通义千问/豆包），返回结果
```
#### 步骤4：测试
运行代码，输入服装相关问题（如“生成牛仔裤小红书文案”），代码会先从你的RAG知识库中检索热搜词/文案模板，再传给大模型生成精准内容，知识库构建完成。

## 核心总结
1. 智能体≠网页，**网页是智能体的演示形式**，课程作业用网页形式最省时、老师最认可，零代码平台一键生成，轻代码10几行代码搞定；
2. RAG知识库构建**核心是整理服装专属数据**，零代码平台拖拽上传即可（首选），轻代码用LangChain+Chroma做，少量Python代码就能实现，均免费、1小时内完成；
3. 无论是哪种方式，构建好的RAG知识库都会让你的通用大模型“秒变服装行业专家”，完全替代微调的效果，且不用等待训练时间，完美贴合你的需求。
